## ИИ-ассистент для обработки запросов закупки светотехнического оборудования

### Описание
Интеллектуальный ассистент принимает на вход файлы с описанием закупки (в формате docx/PDF) -> обрабатывает их -> ищет подходящие позиции в каталоге -> выдает документ с изначальным запросом и подходящими позициями из каталога.


### Системные требования

- Python 3.10-3.12
- 4-8 GB RAM
- Доступ в интернет (для OpenRouter API, Hugging Face)


### Использованные технологии
- **Python**
- **LLM**: OpenAI через LangChain 
- **RAG / векторный поиск**: FAISS
- **Framework**: LangGraph, LangChain (Core, Classic, Community, huggingface)
- **Embeddings**: HuggingFaceEmbeddings
- **Инструменты для повышения качества ответов**: CrossEncoderReranker

### Структура

*   `llm.py` — Основной файл ИИ-агента.
*   `rag.py` — Скрипт для первичной обработки каталога светотехнического оборудования (.xlsx) и создания векторной базы.
*   `requirements.txt` — Информация о версиях библиотек.
*   *База знаний* — Папка для размещения каталога.
*   *Векторная БД* — Локальное хранилище векторных представлений (создается автоматически)

### Архитектура решения

Решение построено на основе RAG (Retrieval-Augmented Generation) и состоит из двух основных модулей:

- rag.py – создание векторного индекса каталога продукции.

- llm.py – обработка закупочного документа: извлечение позиций, поиск соответствий в каталоге и формирование отчёта.

Взаимодействие между модулями: сначала запускается rag.py для построения FAISS-индекса на основе Excel-каталога, затем llm.py использует этот индекс для поиска.

1. **Индексация каталога (rag.py)**

Все листы каталога считываются с помощью pandas -> каждая строка преобразуется в читаемый текстовый фрагмент с колонками «Наименование», характеристики и т.д. (кроме Изображение) -> из строки с текстом создаётся объект Document с метаданными (имя файла, лист, индекс строки) -> полученные документы векторизуются с помощью модели BAAI/bge-m3 и сохраняются в FAISS-индекс для быстрого поиска

2. **Обработка закупочного документа (llm.py)**

- Парсинг файла закупки (docx/pdf) - из документов извлекаются таблицы, которые преобразуются в текст (строки склеиваются пробелами, все ячейки объединяются в одну текстовую строку).

- Выявление позиций и характеристик с помощью LLM - полученный текст передаётся в промпт, который жёстко задаёт структуру ответа в формате JSON. Промпт содержит инструкции: Извлекать только позиции, относящиеся к осветительному оборудованию; для каждой позиции заполнить поля: name, quantity, specifications_raw (все характеристики), specs (структурированный объект с ключами power_w, cct_k, luminous_flux_lm, dimensions_mm); нормализовать числовые значения (из диапазонов брать минимум); ответ модели парсится регулярным выражением, извлекается JSON, и получается список позиций.

- Поиск соответствий в каталоге - для каждой позиции формируется поисковый запрос, который обрабатывается двухступенчатым ретривером: сначала фильтрация по порогу схожести эмбеддингов, потом точное ранжирование оставшихся кандидатов с помощью кросс-энкодер BAAI/bge-reranker-v2-m3. Если кандидатов несколько, LLM выбирает наилучший, анализируя названия и характеристики. Результат для каждой позиции (найденное соответствие, комментарий) сохраняется в pandas.DataFrame и экспортируется в .xlsx в папку results.


### Что не сделано

1. Ассистент не принимает Word документ с расширением .doc. Его нужно предварительно преобразовать в .docx
2. Парсятся только таблицы из закупочных документов. Если запрос будет представлен в виде текста, то ассистент его не распознает


### Реализовано из звездочек

Ассистент принимает на вход PDF-документы.

Добавил наблюдаемость с помощью Langfuse, чтобы иметь возможность фиксировать шаги llm и иметь возможность наладки "приложения" с помощью промтов, так же иметь возможность отслеживать расходы.
